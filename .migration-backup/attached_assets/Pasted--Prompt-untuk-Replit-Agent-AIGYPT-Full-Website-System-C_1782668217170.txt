## SISTEM MEMBER & AKSES

### Model bisnis:
- **Member Mandiri:** Beli kode akses → bisa login → akses semua materi (self-paced)
- **Member Kelas:** Beli paket kelas → bisa login → akses semua materi + info sesi mentoring live

### Alur sistem:
```
User beli (di luar sistem ini) 
→ Admin generate kode akses via Supabase dashboard
→ User dapat kode via WhatsApp/email
→ User buka /login
→ Input email + kode akses
→ Sistem validasi kode di database
→ Jika valid: buat session, redirect ke /dashboard
→ Akses /materi/* terbuka
```

### Supabase Schema:

**Tabel: `access_codes`**
```sql
CREATE TABLE access_codes (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  code VARCHAR(20) UNIQUE NOT NULL,        -- kode unik misal: AIGYPT-A1B2C3
  type VARCHAR(20) NOT NULL,               -- 'mandiri' atau 'kelas'
  batch_number INTEGER,                    -- nomor batch (untuk tipe kelas)
  is_used BOOLEAN DEFAULT FALSE,
  used_by_email VARCHAR(255),
  used_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  expires_at TIMESTAMPTZ                   -- opsional: expired date
);
```

**Tabel: `members`**
```sql
CREATE TABLE members (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  name VARCHAR(255),
  access_code VARCHAR(20) REFERENCES access_codes(code),
  member_type VARCHAR(20) NOT NULL,        -- 'mandiri' atau 'kelas'
  batch_number INTEGER,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  last_login TIMESTAMPTZ
);
```

**Tabel: `materi_progress`**
```sql
CREATE TABLE materi_progress (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  member_email VARCHAR(255) REFERENCES members(email),
  sesi_number INTEGER NOT NULL,           -- 1-6
  is_completed BOOLEAN DEFAULT FALSE,
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(member_email, sesi_number)
);
```

### Auth Flow (tanpa Supabase Auth — custom session):

Gunakan **cookies** untuk session management:
1. User input email + kode akses di `/login`
2. Server validasi: cek kode di tabel `access_codes` (is_used = false atau used_by_email = email ini)
3. Jika valid:
   - Update `access_codes`: set `is_used = true`, `used_by_email = email`, `used_at = now()`
   - Upsert ke tabel `members`
   - Set cookie: `aigypt_session` berisi JSON `{email, memberType, batchNumber}` — enkripsi dengan `jose` library
4. Redirect ke `/dashboard`

Middleware proteksi:
- Route `/dashboard/*` dan `/materi/*`: cek cookie `aigypt_session`
- Jika tidak ada atau expired: redirect ke `/login`

### 6. Middleware (`middleware.ts`)

```typescript
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const session = request.cookies.get('aigypt_session')
  const protectedPaths = ['/dashboard', '/materi']
  
  const isProtected = protectedPaths.some(path => 
    request.nextUrl.pathname.startsWith(path)
  )
  
  if (isProtected && !session) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
  
  return NextResponse.next()
}

export const config = {
  matcher: ['/dashboard/:path*', '/materi/:path*']
}
```

## API ROUTES

### `POST /api/login`
Request body: `{ email: string, code: string }`

Logic:
1. Validasi format email dan kode
2. Query `access_codes` WHERE `code = input_code`
3. Jika tidak ada → return 401 "Kode tidak ditemukan"
4. Jika `is_used = true` AND `used_by_email != input_email` → return 401 "Kode sudah digunakan"
5. Jika valid:
   - Update `access_codes`: `is_used=true`, `used_by_email=email`, `used_at=now()`
   - Upsert `members`: insert atau update `last_login`
   - Generate JWT cookie dengan `jose`: payload `{email, memberType, batchNumber, exp: 30 days}`
   - Set cookie `aigypt_session` (httpOnly, secure, sameSite: lax)
   - Return 200 `{success: true, memberType}`

### `POST /api/logout`
Clear cookie `aigypt_session`, redirect ke `/`

### `POST /api/progress`
Request body: `{ sesiNumber: number }`
Auth: baca dari cookie session
Logic: Upsert `materi_progress` — set `is_completed=true`, `completed_at=now()`
Return: 200 `{success: true}`

### `GET /api/progress`
Auth: baca dari cookie session
Return: array progress peserta `[{sesiNumber, isCompleted, completedAt}]`

## SUPABASE RLS POLICIES

```sql
-- access_codes: hanya service role yang bisa insert/update
ALTER TABLE access_codes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Service role only" ON access_codes
  USING (false);

-- members: hanya bisa baca data sendiri via email match
ALTER TABLE members ENABLE ROW LEVEL SECURITY;

-- materi_progress: hanya bisa baca/tulis data sendiri
ALTER TABLE materi_progress ENABLE ROW LEVEL SECURITY;
```

Semua operasi database dari API routes menggunakan `SUPABASE_SERVICE_ROLE_KEY` (server-side only), bukan anon key. Anon key hanya untuk client-side yang tidak perlu akses database langsung.

## GENERATE KODE AKSES

Tambahkan utility function di `lib/generateCode.ts`:

```typescript
export function generateAccessCode(type: 'mandiri' | 'kelas'): string {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'
  let code = ''
  for (let i = 0; i < 6; i++) {
    code += chars[Math.floor(Math.random() * chars.length)]
  }
  return `AIGYPT-${code}`
}
```

Tambahkan halaman admin sederhana di `/admin/generate` (proteksi dengan basic auth via env variable `ADMIN_PASSWORD`):
- Form: pilih tipe (mandiri/kelas), input batch number (khusus kelas), input jumlah kode yang mau di-generate
- Tombol: `Generate Kode`
- Output: tabel daftar kode yang baru digenerate, bisa di-copy satu per satu atau export ke CSV
- Kode langsung di-insert ke tabel `access_codes` via Supabase service role
