# Prompt — Integrasi Midtrans ke AIGYPT (Webhook + Auto Generate Kode Akses)

Copy-paste prompt ini ke Replit Agent. Ini menambahkan sistem pembayaran Midtrans ke website AIGYPT yang sudah ada — lengkap dengan webhook otomatis yang generate kode akses dan notifikasi WhatsApp setelah pembayaran sukses.

---

## PROMPT

Tambahkan integrasi pembayaran **Midtrans** ke website AIGYPT yang sudah ada. Setelah peserta membayar, sistem harus otomatis: (1) generate kode akses unik, (2) simpan ke database Supabase, (3) kirim kode ke peserta via WhatsApp API atau email. Tanpa intervensi manual dari admin.

PENTING: Ini modifikasi pada proyek yang sudah ada. Jangan bangun ulang dari nol. Pertahankan semua halaman, desain, dan fungsi yang ada.

---

## ARSITEKTUR SISTEM PEMBAYARAN

```
Peserta klik "Daftar Batch 1" di landing page
      ↓
Isi form pendaftaran (nama, email, tipe member)
      ↓
Redirect ke Midtrans Snap (payment page)
      ↓
Peserta bayar (transfer/QRIS/e-wallet)
      ↓
Midtrans kirim webhook ke /api/midtrans/webhook
      ↓
Server validasi signature webhook
      ↓
Generate kode akses unik (format: AIGYPT-XXXXXX)
      ↓
Simpan ke Supabase (tabel access_codes + orders)
      ↓
Kirim kode ke peserta via WhatsApp (Fonnte API)
      ↓
Peserta bisa login dengan email + kode akses
```

## DATABASE — TABEL BARU DI SUPABASE

Jalankan SQL ini di Supabase SQL Editor:

```sql
-- Tabel orders: menyimpan setiap transaksi pembayaran
CREATE TABLE orders (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  order_id VARCHAR(100) UNIQUE NOT NULL,     -- ID unik untuk Midtrans
  email VARCHAR(255) NOT NULL,
  name VARCHAR(255) NOT NULL,
  phone VARCHAR(20),                          -- untuk kirim WA
  member_type VARCHAR(20) NOT NULL,           -- 'mandiri' atau 'kelas'
  batch_number INTEGER DEFAULT 1,
  amount INTEGER NOT NULL,                    -- harga dalam rupiah
  status VARCHAR(20) DEFAULT 'pending',       -- pending/paid/failed/expired
  midtrans_transaction_id VARCHAR(255),
  access_code VARCHAR(20),                    -- kode yang digenerate
  paid_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Index untuk performa
CREATE INDEX orders_email_idx ON orders(email);
CREATE INDEX orders_order_id_idx ON orders(order_id);
CREATE INDEX orders_status_idx ON orders(status);

-- RLS
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;
-- Hanya service role yang bisa akses (semua dari serverless function)
```

Update tabel `access_codes` yang sudah ada — tambahkan relasi ke orders:
```sql
ALTER TABLE access_codes ADD COLUMN IF NOT EXISTS order_id VARCHAR(100) REFERENCES orders(order_id);
ALTER TABLE access_codes ADD COLUMN IF NOT EXISTS batch_number INTEGER DEFAULT 1;
```

### 2. Serverless Function: Buat Transaksi (`api/create-transaction.ts`)

```typescript
import type { VercelRequest, VercelResponse } from '@vercel/node'
import { createClient } from '@supabase/supabase-js'
import crypto from 'crypto'

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
)

export default async function handler(req: VercelRequest, res: VercelResponse) {
  if (req.method !== 'POST') {
    return res.status(405).json({ error: 'Method not allowed' })
  }

  const { name, email, phone, memberType } = req.body

  // Validasi input
  if (!name || !email || !phone || !memberType) {
    return res.status(400).json({ error: 'Semua field wajib diisi' })
  }

  // Cek apakah email sudah pernah bayar (hindari double order)
  const { data: existingOrder } = await supabase
    .from('orders')
    .select('id, status, access_code')
    .eq('email', email)
    .eq('status', 'paid')
    .single()

  if (existingOrder) {
    return res.status(400).json({
      error: 'Email ini sudah terdaftar. Silakan login dengan kode akses yang sudah dikirimkan.'
    })
  }

  // Generate order ID unik
  const orderId = `AIGYPT-${Date.now()}-${Math.random().toString(36).substr(2, 6).toUpperCase()}`

  // Tentukan harga
  const amount = memberType === 'kelas'
    ? parseInt(process.env.PRICE_KELAS || '300000')
    : parseInt(process.env.PRICE_MANDIRI || '150000')

  // Simpan order ke database (status: pending)
  await supabase.from('orders').insert({
    order_id: orderId,
    email,
    name,
    phone,
    member_type: memberType,
    batch_number: 1,
    amount,
    status: 'pending'
  })

  // Buat transaksi Midtrans
  const serverKey = process.env.MIDTRANS_SERVER_KEY!
  const isProduction = process.env.MIDTRANS_IS_PRODUCTION === 'true'
  const baseUrl = isProduction
    ? 'https://app.midtrans.com'
    : 'https://app.sandbox.midtrans.com'

  const midtransResponse = await fetch(`${baseUrl}/snap/v1/transactions`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Basic ${Buffer.from(serverKey + ':').toString('base64')}`
    },
    body: JSON.stringify({
      transaction_details: {
        order_id: orderId,
        gross_amount: amount
      },
      customer_details: {
        first_name: name,
        email: email,
        phone: phone
      },
      item_details: [{
        id: memberType,
        price: amount,
        quantity: 1,
        name: memberType === 'kelas'
          ? 'AIGYPT Class Batch 1 — Member Kelas'
          : 'AIGYPT Class Batch 1 — Member Mandiri'
      }],
      callbacks: {
        finish: `${req.headers.origin}/sukses`
      }
    })
  })

  const midtransData = await midtransResponse.json()

  if (!midtransData.token) {
    return res.status(500).json({ error: 'Gagal membuat transaksi. Coba lagi.' })
  }

  return res.status(200).json({
    snap_token: midtransData.token,
    order_id: orderId
  })
}
```

### 3. Serverless Function: Webhook Midtrans (`api/midtrans/webhook.ts`)

Ini jantung sistemnya. Dipanggil Midtrans setiap ada update transaksi.

```typescript
import type { VercelRequest, VercelResponse } from '@vercel/node'
import { createClient } from '@supabase/supabase-js'
import crypto from 'crypto'

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
)

// Generate kode akses unik format AIGYPT-XXXXXX
function generateAccessCode(): string {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'
  let code = ''
  for (let i = 0; i < 6; i++) {
    code += chars[Math.floor(Math.random() * chars.length)]
  }
  return `AIGYPT-${code}`
}

// Validasi signature dari Midtrans (WAJIB untuk keamanan)
function validateSignature(
  orderId: string,
  statusCode: string,
  grossAmount: string,
  serverKey: string,
  signatureKey: string
): boolean {
  const hash = crypto
    .createHash('sha512')
    .update(`${orderId}${statusCode}${grossAmount}${serverKey}`)
    .digest('hex')
  return hash === signatureKey
}

// Kirim WhatsApp via Fonnte
async function sendWhatsApp(phone: string, message: string): Promise<void> {
  await fetch('https://api.fonnte.com/send', {
    method: 'POST',
    headers: {
      'Authorization': process.env.FONNTE_TOKEN!
    },
    body: new URLSearchParams({
      target: phone,
      message: message,
      countryCode: '62'
    })
  })
}

export default async function handler(req: VercelRequest, res: VercelResponse) {
  if (req.method !== 'POST') {
    return res.status(405).json({ error: 'Method not allowed' })
  }

  const notification = req.body
  const {
    order_id,
    transaction_status,
    fraud_status,
    gross_amount,
    status_code,
    signature_key,
    transaction_id
  } = notification

  // 1. Validasi signature (keamanan — pastikan request dari Midtrans)
  const isValid = validateSignature(
    order_id,
    status_code,
    gross_amount,
    process.env.MIDTRANS_SERVER_KEY!,
    signature_key
  )

  if (!isValid) {
    console.error('Invalid Midtrans signature')
    return res.status(401).json({ error: 'Invalid signature' })
  }

  // 2. Ambil data order dari database
  const { data: order, error } = await supabase
    .from('orders')
    .select('*')
    .eq('order_id', order_id)
    .single()

  if (error || !order) {
    return res.status(404).json({ error: 'Order not found' })
  }

  // 3. Cek apakah sudah diproses sebelumnya (idempotency)
  if (order.status === 'paid') {
    return res.status(200).json({ message: 'Already processed' })
  }

  // 4. Tentukan status transaksi
  const isPaid =
    transaction_status === 'capture' && fraud_status === 'accept' ||
    transaction_status === 'settlement'

  const isFailed =
    transaction_status === 'cancel' ||
    transaction_status === 'deny' ||
    transaction_status === 'expire'

  // 5. Proses jika pembayaran sukses
  if (isPaid) {
    // Generate kode akses unik (pastikan tidak duplikat)
    let accessCode = ''
    let isUnique = false
    while (!isUnique) {
      accessCode = generateAccessCode()
      const { data: existing } = await supabase
        .from('access_codes')
        .select('code')
        .eq('code', accessCode)
        .single()
      if (!existing) isUnique = true
    }

    // Simpan kode akses ke tabel access_codes
    await supabase.from('access_codes').insert({
      code: accessCode,
      type: order.member_type,
      batch_number: order.batch_number,
      is_used: false,
      order_id: order.order_id
    })

    // Update order: status paid, simpan kode dan transaction_id
    await supabase
      .from('orders')
      .update({
        status: 'paid',
        access_code: accessCode,
        midtrans_transaction_id: transaction_id,
        paid_at: new Date().toISOString()
      })
      .eq('order_id', order_id)

    // Kirim kode akses ke peserta via WhatsApp
    const memberLabel = order.member_type === 'kelas' ? 'Member Kelas' : 'Member Mandiri'
    const pesertaMessage =
      `Assalamualaikum ${order.name}! 🎉\n\n` +
      `Pembayaran AIGYPT Class Batch 1 kamu sudah dikonfirmasi.\n\n` +
      `Paket: *${memberLabel}*\n` +
      `Kode Akses: *${accessCode}*\n\n` +
      `Cara login:\n` +
      `1. Buka aigypt.id/login\n` +
      `2. Masukkan email: ${order.email}\n` +
      `3. Masukkan kode akses di atas\n\n` +
      `Simpan kode ini baik-baik ya. Selamat belajar! 🚀\n\n` +
      `— Tim AIGYPT`

    await sendWhatsApp(order.phone, pesertaMessage)

    // Kirim notifikasi ke admin
    const adminMessage =
      `✅ *Pembayaran Baru AIGYPT*\n\n` +
      `Nama: ${order.name}\n` +
      `Email: ${order.email}\n` +
      `WA: ${order.phone}\n` +
      `Paket: ${memberLabel}\n` +
      `Amount: Rp ${parseInt(gross_amount).toLocaleString('id-ID')}\n` +
      `Kode: ${accessCode}\n` +
      `Order ID: ${order_id}`

    await sendWhatsApp(process.env.ADMIN_WA_NUMBER!, adminMessage)

  } else if (isFailed) {
    // Update status order jadi failed/expired
    await supabase
      .from('orders')
      .update({ status: transaction_status })
      .eq('order_id', order_id)
  }

  // Selalu kembalikan 200 ke Midtrans
  return res.status(200).json({ message: 'OK' })
}
```
