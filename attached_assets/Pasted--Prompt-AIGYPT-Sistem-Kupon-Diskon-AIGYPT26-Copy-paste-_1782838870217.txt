## LANGKAH 1 — Tabel Database Baru

Tambahkan tabel baru di Supabase untuk tracking penggunaan kupon:

```sql
CREATE TABLE coupon_usage (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  coupon_code VARCHAR(50) NOT NULL,
  email VARCHAR(255) NOT NULL,
  order_id VARCHAR(100) REFERENCES orders(order_id),
  discount_amount INTEGER NOT NULL,
  used_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(coupon_code, email)
);

ALTER TABLE coupon_usage ENABLE ROW LEVEL SECURITY;
```

UNIQUE(coupon_code, email) memastikan satu email tidak bisa memakai kupon yang sama dua kali, divalidasi di level database, bukan hanya di frontend.

Tambahkan juga kolom pada tabel orders yang sudah ada untuk mencatat diskon yang diterapkan:
```sql
ALTER TABLE orders ADD COLUMN IF NOT EXISTS coupon_code VARCHAR(50);
ALTER TABLE orders ADD COLUMN IF NOT EXISTS discount_amount INTEGER DEFAULT 0;
ALTER TABLE orders ADD COLUMN IF NOT EXISTS final_amount INTEGER;
```

final_amount adalah nominal setelah dikurangi diskon, ini yang ditampilkan sebagai jumlah yang harus dibayar di halaman QRIS.

## LANGKAH 2 — Konfigurasi Kupon (Sumber Kebenaran Tunggal)

Buat file konfigurasi kupon agar aturan kupon mudah dikelola di satu tempat:

```typescript
// lib/coupons.ts
export interface Coupon {
  code: string
  discountAmount: number
  applicableTo: 'kelas' | 'mandiri' | 'all'
  validUntil: string
  description: string
}

export const ACTIVE_COUPONS: Coupon[] = [
  {
    code: 'AIGYPT26',
    discountAmount: 51000,
    applicableTo: 'kelas',
    validUntil: '2026-07-06T23:59:59+07:00',
    description: 'Diskon spesial pendaftaran Batch 1'
  }
]

export function validateCoupon(
  code: string,
  memberType: 'kelas' | 'mandiri'
): { valid: boolean; coupon?: Coupon; error?: string } {
  const coupon = ACTIVE_COUPONS.find(
    (c) => c.code.toUpperCase() === code.trim().toUpperCase()
  )

  if (!coupon) {
    return { valid: false, error: 'Kode kupon tidak ditemukan' }
  }

  const now = new Date()
  const expiry = new Date(coupon.validUntil)
  if (now > expiry) {
    return { valid: false, error: 'Kode kupon sudah tidak berlaku' }
  }

  if (coupon.applicableTo !== 'all' && coupon.applicableTo !== memberType) {
    return {
      valid: false,
      error: 'Kupon ini hanya berlaku untuk Member Kelas'
    }
  }

  return { valid: true, coupon }
}
```

### Logic Validasi Kupon (Client-side, untuk UX cepat)

Saat tombol "Terapkan" diklik:
1. Panggil fungsi validateCoupon(code, selectedMemberType) dari lib/coupons.ts
2. Jika valid secara aturan umum, lakukan pengecekan tambahan ke server (karena pengecekan sudah pernah dipakai email ini atau belum butuh akses database) via API route baru api/coupons/check.ts
3. Tampilkan hasil validasi (sukses dengan nominal diskon, atau pesan error spesifik)

### API Route Baru, Cek Pemakaian Kupon

```typescript
// api/coupons/check.ts
import type { VercelRequest, VercelResponse } from '@vercel/node'
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
)

export default async function handler(req: VercelRequest, res: VercelResponse) {
  if (req.method === 'OPTIONS') return res.status(200).end()
  if (req.method !== 'POST') {
    return res.status(405).json({ error: 'Method not allowed' })
  }

  const { code, email } = req.body

  if (!code || !email) {
    return res.status(400).json({ valid: false, error: 'Data tidak lengkap' })
  }

  const { data: existingUsage } = await supabase
    .from('coupon_usage')
    .select('id')
    .eq('coupon_code', code.toUpperCase())
    .eq('email', email.toLowerCase())
    .single()

  if (existingUsage) {
    return res.status(200).json({
      valid: false,
      error: 'Kupon ini sudah pernah kamu gunakan'
    })
  }

  return res.status(200).json({ valid: true })
}
```

### Update Logic Saat Form Disubmit

Jika kupon diterapkan dan valid (lolos pengecekan client-side dan server-side):
1. Hitung finalAmount = baseAmount dikurangi discountAmount
2. Simpan coupon_code, discount_amount, dan final_amount ke tabel orders saat membuat order baru
3. Catat penggunaan ke tabel coupon_usage (coupon_code, email, order_id, discount_amount)

PENTING: Lakukan validasi kupon SEKALI LAGI di server (bukan hanya mengandalkan validasi client-side) sebelum menyimpan order, untuk mencegah manipulasi nominal dari sisi client. Validasi akhir ini sebaiknya terjadi di API route yang menangani pembuatan order (api/orders/create.ts atau sejenisnya sesuai struktur yang sudah ada untuk flow QRIS).

## REQUIREMENTS TEKNIS

1. Validasi kupon WAJIB dilakukan di server (bukan hanya client) sebelum order final dibuat, untuk mencegah manipulasi nominal
2. Constraint UNIQUE di database (coupon_code, email) menjadi pengaman utama terhadap pemakaian ganda, jangan hanya mengandalkan pengecekan di kode aplikasi
3. Tanggal kedaluwarsa kupon dibandingkan menggunakan timezone yang konsisten (WIB, UTC+7)
4. Semua teks Bahasa Indonesia, panggilan "kamu"
5. Pesan error kupon jelas dan spesifik (kupon tidak ditemukan, sudah kedaluwarsa, tidak berlaku untuk paket ini, atau sudah pernah dipakai), bukan pesan generik
6. npm run build harus sukses
